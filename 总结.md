# U-CDMR-Flow-Residual+ 当前方案总结

## 1. 当前定位

当前工程只保留新主线：

```text
U-CDMR-Flow-Residual+
```

目标不是用 FLUX / Stable Diffusion 生成整张自然图，而是做任务专用裂缝合成：

```text
真实 normal 背景
+ 合理 crack mask
+ mask-gated crack residual
= synthetic crack image + exact synthetic mask
```

核心公式：

```text
I_syn = I_normal + gate(M_syn) * Delta_crack
Y_syn = M_syn
```

这样做的重点是：背景不重绘、mask 和图像强绑定、dphone 保持原生分辨率，最终用下游分割指标判断合成数据有没有价值。

```text

```

dphone 不全局 resize、不全局 crop。训练可以 tile，但生成、保存、评估都回到 native canvas。

## 3. 模型路线

### Teacher / Downstream Segmenter

```text
image
  -> ImageNet-pretrained ResNet encoder
  -> segmentation decoder
  -> crack probability mask
```

Teacher 不是最终裁判，只提供辅助评分；最终判断看真实 held-out test 上的分割结果。

### Residual Flow Matching 主生成器

训练时从真实 crack 图构造 pseudo-normal：

```text
I_crack + M_raw
  -> erase crack
  -> I_ctx
```

Residual flow 学习的是裂缝 residual 的 velocity field：

```text
x1 = gate(M_real) * (I_crack - I_ctx)
x0 = gate(M_real) * gaussian_noise
x_t = (1 - t) * x0 + t * x1
v_target = x1 - x0

FlowModel(x_t, t, I_ctx, mask_condition, domain, style)
  -> v_pred
```

生成时用 ODE sampling 得到 `Delta_flow`，再融合：

```text
I_syn = I_normal + M_gate * Delta_flow
Y_syn = M_syn
```

loss 里：

```text
M_band: 主要重建区域
M_gate: 残差注入和泄漏约束区域
outside gate: 背景保持不变
```

### Mask Generator

第一版 mask 来源是 real mask bank。

可选 Mask Flow 不是生成整张 mask 图，而是生成 mask descriptor：

```text
center_x / center_y / area / bbox / orientation / thickness / topology
```

然后用真实 mask template 渲染回 full-resolution mask：

```text
descriptor flow sample
  -> choose similar real template
  -> resize / rotate / place
  -> full native-resolution mask
```

这比直接生成像素 mask 稳，因为裂缝细结构来自真实 template。

## 4. 当前完整流程

```text
manifest_merged.csv
  -> filter broken
  -> video-level train/val/test split
  -> LabelMe crack annotation to masks
  -> pseudo-normal quality check
  -> train teacher segmenter
  -> train residual flow
  -> optional descriptor-level Mask Flow
  -> generate synthetic on train normal backgrounds
  -> filter synthetic with residual / leakage / IoU / optional teacher scores
  -> train downstream segmenter
  -> evaluate on real val/test only
```

Synthetic 只从 train split 的 real normal 背景生成。val/test 不参与生成和过滤阈值选择。

## 5. 当前不用什么

当前主线不使用：

```text
FLUX
Stable Diffusion
full-image DiT
VAE latent diffusion
paired crack-normal pixel residual supervision
```

原因是这些路线容易改变背景纹理和像素尺度，而你的任务需要保留接触线图像的真实几何和像素测量意义。

```bash
python -m pip install -e .
# or
export PYTHONPATH="$PWD/src:${PYTHONPATH:-}"
```

数据准备：

```bash

ucdmr_plus_prepare_splits

ucdmr_plus_prepare_pseudo_normal --split train
```

训练和生成：

```bash
ucdmr_plus_train_teacher
ucdmr_plus_train_residual_flow
ucdmr_plus_train_mask_descriptor_flow
ucdmr_plus_sample_masks
ucdmr_plus_generate_synthetic
ucdmr_plus_filter_synthetic
ucdmr_plus_train_downstream
ucdmr_plus_eval_downstream
```

```bash

```

非 `sbatch` 的生成 / eval / mask-flow 脚本默认要求在 Slurm allocation 里运行，避免误把重任务跑在 login node。
